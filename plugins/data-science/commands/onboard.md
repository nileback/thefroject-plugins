# /onboard — Generate Onboarding Guide

Create an onboarding guide for new team members.

1. Analyze the project structure and tech stack
2. Generate:
   - Environment setup guide (step-by-step)
   - Architecture overview (key files, data flow)
   - Development workflow (branch, test, deploy)
   - Key conventions and patterns used
   - Common tasks with examples
3. Save to `docs/onboarding.md` or similar

Usage: `/onboard`
