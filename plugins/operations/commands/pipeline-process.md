# /pipeline-process: Process Improvement Pipeline

Run the full process improvement pipeline.

## Steps

### 1. Process mapping
Document current-state processes with steps, owners, handoffs, and bottlenecks.

Output: process map.

### 2. SOP writing
Write clear standard operating procedures from the process maps. Include decision trees and exception handling.

Output: SOP documents.

### 3. Risk assessment
Identify and score risks with mitigation strategies for each process.

Output: risk register.

### 4. Vendor review
Review vendor dependencies in the process. Assess performance, cost, and alternatives.

Output: vendor assessment.

## How to use

Run: `/pipeline-process [process name]`
