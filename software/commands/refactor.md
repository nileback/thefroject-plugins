# /refactor — Refactor Code

Refactor the specified code safely.

1. Read the code and identify refactoring opportunities
2. Verify tests exist (write them first if not)
3. Make one change at a time, verifying tests pass
4. Focus on: reducing duplication, improving naming, simplifying logic
5. Never change behavior — only structure

Usage: `/refactor [file-path-or-description]`
