# /debug — Debug Issue

Systematically debug the described issue.

1. **Reproduce** — Understand the exact symptoms and steps
2. **Isolate** — Narrow down to the relevant file/function/line
3. **Analyze** — Read surrounding code, trace data flow
4. **Hypothesize** — Form a theory about root cause
5. **Verify** — Test the hypothesis with targeted investigation
6. **Fix** — Apply minimal fix addressing root cause
7. **Prevent** — Add a test to catch regression

Usage: `/debug [description of the issue]`
