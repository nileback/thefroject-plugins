# /document — Generate Documentation

Generate documentation for the specified code.

1. Read the source code
2. Generate:
   - Function/class docstrings
   - README section for the module
   - Usage examples (runnable)
   - API reference if applicable
3. Follow existing documentation style in the project
4. Focus on "why" and "how to use", not just "what"

Usage: `/document [file-or-directory]`
